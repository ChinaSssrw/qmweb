# 企盟小程序 — AI协作规则

> 为AI（开发者助手）设定编码协作规则，确保输出一致、可维护。

## 通用原则

1. **文档不是交付物，是AI协作的上下文控制系统** — 每次开工前先读 vision.md
2. **代码先行于注释**：写清晰自文档化的代码，注释只解释 why，不解释 what
3. **所有新功能必须同步生成对应的单元测试**
4. **修改代码时，同时更新对应的文档**（本文档体系）
5. **永远不删除或跳过现有测试**，除非明确指示
6. **每次修改后，主动列出需要运行的验证命令**

## 后端开发规则

- FastAPI 路由命名：`/api/{resource}/{action}`
- 所有API返回统一格式：`{"code": 0, "data": ..., "message": "ok"}`
- 数据库操作使用 SQLAlchemy ORM
- 敏感操作（支付、审核）必须使用数据库事务
- 语音/文档解析结果必须经用户确认后才写入数据库
- 错误码规范：0=成功, 1001=参数错误, 1002=权限不足, 2001=支付失败, 3001=数据库错误

## 前端开发规则

- 使用微信小程序原生框架
- 页面组件化：每个独立模块封装为自定义组件
- API调用统一封装在 `utils/request.js` 中
- 名片语音输入：录音API → 上传 → 轮询结果
- 错误处理：网络异常、API错误统一弹Toast

## 测试规则

- 每次提供代码修改后，必须同时提供对应的单元测试
- 改动可能影响其他模块时，主动列出需重新运行的测试
- 修复Bug时：先生成能复现Bug的测试用例，修复后该用例通过
- 测试命名使用业务语义：`should_lock_account_after_3_failed_attempts`
- 覆盖率目标：新代码行覆盖 >= 80%，分支覆盖 >= 70%
