# 模块1：数字名片 📇

> 用户的核心身份载体

## 名片构成（5个信息板块）

| 板块 | 内容 |
|------|------|
| 基本信息 | 姓名、头像、公司、职位、地区 |
| 联系方式 | 手机（脱敏）、微信（脱敏）、邮箱（脱敏） |
| 身份标签 | 行业、职能、标签关键词 |
| 个人简介 | 一句话身份描述、详细自我介绍 |
| 社交链接 | 公众号、视频号、其他平台（可选） |

## 录入方式（三种并行）

1. **语音输入**（核心体验）：调用微信录音API → 语音转文字 → NLP分类填充到对应板块
2. **文档读取**：上传Word/PDF/TXT → 后端解析提取 → 用户确认后写入
3. **手动输入**：点击编辑框直接填写

> 三者可互补使用，同一字段以最后一次修改为准。
> 语音和文档输入的解析结果必须经用户确认后才写入数据库。

## 隐私层级

| 层级 | 可见范围 |
|------|----------|
| 公开 | 姓名、头像、身份标签——所有人可见 |
| 好友可见 | 手机号（脱敏）、微信（脱敏）、邮箱、简介 |
| 付费好友可见 | 完整联系方式、详细自我介绍 |
| 仅自己可见 | 编辑中未完成的信息、内部备注 |

## 名片传播

- 用户可将名片分享到微信群/好友，生成名片卡片
- 别人点击后看到公开字段 + "加好友"按钮
- 点击加好友 → 弹出注册/登录引导（注明推荐人信息）

## 相关API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/profile | 获取当前用户名片 |
| PUT | /api/profile | 手动编辑更新名片 |
| POST | /api/profile/voice | 语音输入（音频→转文字→NLP分类） |
| POST | /api/profile/document | 文档上传（文档→解析→确认写入） |
