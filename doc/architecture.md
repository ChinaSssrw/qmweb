# 企盟小程序 — 技术架构

> 当前决策：MVP优先，不引入微服务

## 技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| 后端框架 | Python + FastAPI | 异步性能好，Swagger自动文档，AI集成方便 |
| 数据库 | MySQL 8.0 | 关系型数据，事务支持，微信生态成熟 |
| 缓存 | Redis | 会话管理、活动状态、防骚扰计数 |
| 语音处理 | 阿里云语音识别API | 语音→文字，NLP分类填充名片 |
| 文档解析 | python-docx / PyMuPDF | Word/PDF/TXT提取关键信息 |
| 支付 | 微信支付JSAPI | 小程序内支付 |
| 部署 | 阿里云ECS + OSS | 后端+静态资源+文件存储 |
| AI工具 | 百炼/通义千问API | 活动策划、文案、海报生成 |

## 架构原则

1. **MVP优先**：第一阶段用最简架构，不引入微服务
2. **AI优先**：语音→文字、文档解析、内容生成等AI能力作为基础设施
3. **数据一致**：关键操作（审核+支付）使用数据库事务
4. **安全优先**：微信登录凭证、支付签名、用户隐私严格保护

## 部署架构（MVP）

```
用户（微信小程序）
     ↕ HTTPS
阿里云ECS（FastAPI + Gunicorn + Uvicorn）
     ↕
阿里云RDS MySQL ←→ Redis（缓存/会话）
     ↕
阿里云OSS（名片文档/活动图片/空间照片）
     ↕
外部API：微信支付 / 阿里云语音识别 / 百炼AI
```
