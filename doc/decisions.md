# 企盟小程序 — 架构决策记录（ADR）

> 记录重要架构决策及其理由，防止AI或后续开发者反复推翻已定方案。

## ADR-001：后端选型

**决策：** Python + FastAPI
**原因：** 语音转文字和文档解析在Python生态成熟；FastAPI异步性能好；AI集成方便
**日期：** 2026-05-12

## ADR-002：推荐注册必选

**决策：** 注册必须绑定推荐人
**原因：** 保证用户质量，建立推荐链，冷启动依赖种子用户裂变
**日期：** 2026-05-12

## ADR-003：付费审核双审制

**决策：** 推荐人审核 + 管理员审核
**原因：** 推荐人确认身份真实性，管理员确认资质合规，双重保障
**日期：** 2026-05-12

## ADR-004：支付走微信JSAPI

**决策：** 小程序内微信支付
**原因：** 微信小程序官方支付渠道，用户体验一致，资金安全
**日期：** 2026-05-12

## ADR-005：语音输入优先

**决策：** 语音 > 文档 > 手动（录入优先级）
**原因：** 降低录入门槛，提升名片完成率，差异化体验
**日期：** 2026-05-12

## ADR-006：文档体系（Vibe Coding）

**决策：** 采用Vibe Coding文档框架，doc/目录下按vision/design/modules/等分层组织
**原因：** 让AI（开发者助手）快速掌握项目上下文，减少反复解释
**日期：** 2026-05-12
