# 企盟小程序 — 测试策略

> Vibe Coding中的测试是由AI辅助生成、人工审核+自动化执行的连续反馈系统。

## 测试分层

| 层级 | 覆盖范围 | 工具 |
|------|----------|------|
| 单元测试 | 领域模型、工具函数、关键算法 | Jest（前端）/ Pytest（后端） |
| 集成测试 | 数据库访问层、外部API适配器 | Pytest + SQLAlchemy |
| 端到端测试 | 核心用户旅程（注册/发起活动/付费） | 微信开发者工具 + 自动化 |
| 契约测试 | API接口与api-contract.md一致性 | Postman / Swagger |

## 覆盖率目标

- 行覆盖率 >= 80%（新代码必须达标）
- 分支覆盖率 >= 70%

## 统一测试命令

```
pytest                     # 全量测试
pytest tests/unit          # 仅单元测试
pytest tests/integration   # 仅集成测试
npm run test:e2e           # 端到端测试
```

## AI参与测试的方式

- 所有新功能/修改 **必须** 由AI同步生成对应的单元测试
- AI生成的测试代码需经过人工审查，确认无逻辑错误
- 每次对话末尾，AI必须主动建议可以运行哪些测试来验证变更
- 修复Bug时：先生成能复现Bug的测试用例，修复后该用例通过

## 测试数据与Mock

- 使用 `mocks-and-fixtures.md` 中定义的测试数据
- 禁止在测试中连接真实第三方服务，一律使用Mock或本地模拟

## 测试与AI的闭环工作流

```
需求/任务(tasks.md)
       ↓
AI 生成实现 + 单元测试
       ↓
运行测试（自动/手动触发）
       ↓
失败 → 测试结果 + 错误信息反馈给 AI → 修正代码
成功 → 代码审查 + 合并
       ↓
更新测试用例清单 / 记录决策(decisions.md)
```
